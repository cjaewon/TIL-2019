# Git 명령어 및 사용방법 공부

### 🗂 현재 폴더에 깃 설정하기

**클론을 받은 경우에는 필요하지않음!**  
```git init``` 을 통해서 현재 폴더를 깃 폴더로 설정해 깃 명령어가 사용 가능하다.

`git remote add origin https://github.com/cjaewon/thisisgithub.git`  
을 통하여 연결한다.

### 🔃 최신 소스코드로 업데이트

`git pull origin master` 을 통하여 최신 코드로 업데이트 할 수 있다.
강제로 최신 소스코드로 업데이트 하고 싶다면 이것도 `-f` flag를 붙여주면 된다. 

`git fetch` - 추가 예정

### ⬆️ 깃허브에 올리기

`git add .` - 깃이 파일들을 추척한다 ( ex 수정된 파일, 삭제된 파일, 새로 생긴 파일 등  )   
만약 강제로 할려면 `-f` flag를 붙혀주면 된다

`git commit -m "Message"` - 변경상황을 내 컴퓨터에 저장한다  
만약 실수로 커밋했다면 수정 후에--amend flag 를 붙여주면 된다.

`git push origin master` - 깃허브에 업로드한다.

### 🌮 branch 생성

`git branch [브랜치명]` - 브랜치를 생성한다.  
`git checkout [브랜치명]` - 브랜치에 접속한다.

`git branch [브랜치명] -b` - 이 두개를 동시에 할려면 -b flag를 붙여준다.  
삭제는 -d flag를 붙여준다

그리고 원격 저장소 (깃허브)에 반영하기 위해서는  
`git push origin :[브랜치명]`을 해준다. ( 콜론을 꼭 붙여주어야한다 )

### ➕ 파일 병합
다른 브랜치에서 개발 된 것을 합칠때 사용한다.  
먼저 `git checkout master`을 입력하여 master 브랜치를 사용하게 한다.  
또는 현재 브랜치에 합치고자 하는 브랜치명을 사용

`git merge`  
서로 다른 파일을 수정했을때는 자동으로 깃이 합쳐준다  
같은 파일이여도 다른 줄을 수정했다면 깃이 자동으로 합쳐준다

하지만 같은 줄을 수정했을 경우에는 충돌이 나게되는데 이때는 깃이 자동으로 병합할 수 없다.  
그러므로 개발자가 고쳐주어야한다.  

`git status`를 사용해서 충돌나는 부분을 확인하자  
`<<<<<<< HEAD`  ~ `=========` 는 마스터 브랜치의 코드이고
`=========` ~ `>>>>>>>>>> 브랜치명`은 마스터 이외의 브랜치 코드이다

그러면 코드를 수정 후 커밋한다.




`git rebase`



### 😞 깃무시 ? .gitignore

git이 자동으로 추가할 필요가 없는 폴더, 파일이 있다  
이럴때에는 `.gitignore` 에 추가하면 된다.
